    Spinxドキュメントの作成方法¶

    Python仮想環境の作成¶

    はじめに、SphinxライブラリをPython環境にインストールします。他のプロジェクトとライブラリの競合を避けるために仮想環境の利用を推奨します。仮想環境は次のコマンドで作成できます。

    python -m venv YOUR_VIRTUAL_ENVIRONMENT_PATH

    例えば、"py310"という名前の仮想環境を"~/.venv"ディレクトリに作成する場合は次のコマンドを実行します。

    python -m venv ~/.venv/py310

    仮想環境を利用するためにはアクティベートする必要があります。仮想環境のアクティベートは次のコマンドで行います。

    source ~/.venv/py310/bin/activate

    もしWindowsのコマンドプロンプトを利用している場合は上記の代わりに次のコマンドを実行してください。

    ~/.venv/py310/Scripts/activate

    仮想環境のアクティベートが正しく行われると、コマンド行の先頭に仮想環境名が表示されるようになります。

    (py310) shun@MacBook-Pro mynote %

    Sphinxのインストール¶

    Sphinxは次のコマンドでインストールできます。

    python -m pip install sphinx

    Sphinxのサードパティの拡張機能は色々あり、それらを使用するにはライブラリパッケージをインストールする必要があります。拡張機能はドキュメントの見た目を変化させるテーマやフローチャート作成ツールなどがあります。例えば、このドキュメントでも使用している"sphinx-material"というテーマをインストールするには次のコマンドを実行します。

    python -m pip install sphinx-material

    プロジェクトディレクトリの作成¶

    はじめに、次の構成のディレクトリを作成してください。

    PROJECT_ROOT
`- src

    "src"ディレクトリで次のコマンドを実行すると"docs"というSphinxのプロジェクトディレクトリを作成できます。

    sphinx-quickstart -q -p PROJECT_NAME -a AUTHOR_NAME ./docs

    ここで、プロジェクトディレクトリは以下のような構成になっています。

    PROJECT_ROOT
`- src
   `- docs
      |- _build
      |- _static
      |- _templates
      |- Makefile
      |- make.bat
      |- conf.py
      `- index.rst

    最初のドキュメントのビルド¶

    このセクションではHTMLドキュメントを作成します。ドキュメントファイルはGitHub Pagesにアップロードできるように、プロジェクトディレクトリ直下の"docs"に出力します。

    Makefileを編集して"docs"ディレクトリにドキュメントファイルをコピーする処理を追加します。

    + html: Makefile
+     @$(SPHINXBUILD) -M $@ "$(SOURCEDIR)" "$(BUILDDIR)" $(SPHINXOPTS) $(O)
+    if [ -e ../../docs ]; then rm -rf ../../docs; fi
+    cp -r ./_build/html ../../docs
+    touch ../../docs/.nojekyll

%: Makefile
    @$(SPHINXBUILD) -M $@ "$(SOURCEDIR)" "$(BUILDDIR)" $(SPHINXOPTS) $(O)

    "src/docs"ディレクトリで次のコマンドを実行しましょう。"src/docs/_build/html"ディレクトリと"docs"ディレクトリにHTMLドキュメントファイルが出力されます。

    make html

    ドキュメントファイルの出力後、出力ディレクトリにある"index.html"をGoogle ChromeなどのWebブラウザで開きましょう。

    GitHub Pagesでのドキュメント公開¶

    GitHub Pagesサービスを使えば簡単にドキュメントを公開できます。ドキュメントの公開は次の5ステップで行えます。

    1. GitHubにプロジェクトリポジトリを作成する。

    2. "docs"ディレクトリにドキュメントファイルをビルドする。

    3. "docs"ディレクトリをステージングして、コミットする。

    4. GitHubのリモートリポジトリにプッシュする。

    5. GitHub Pagesの設定を変更する。

    GitHub Pagesの設定¶

    以下に、5ステップ目の操作の詳細を記載します。

    1. GitHubのプロジェクトリポジトリのページにアクセスして"settings"を開く。

    2. サイドバーメニューから"Code and automation"の"Pages"を選択する。

    3. "Build and deployment"セクションの"Branch"で、branchを"main"に、folderを"/docs"に変更する。

    4. branchとfolderの設定後にsaveボタンを押して変更を反映する。

    5. 設定後、GitHub Pagesのサイトがビルドされ、URLと"Visit site"ボタンが設定ページの上部に表示される。アクセスして確認してみましょう!

    ドキュメントの国際化¶

    ドキュメントをSphinx公式のとおりに国際化します。

    はじめに、"sphinx-intl"パッケージをpipコマンドでインストールします。

    python -m pip install sphinx-intl

    次に、conf.pyを編集して設定を追加します。

    + locale_dirs = ['locale/']
+ gettext_compact = False

    そして"src/docs"ディレクトリで次のコマンドを実行してPOTファイルを作成します。POTファイルは"src/docs/build"ディレクトリに出力されます。

    make gettext

    POファイルを次のコマンドで作成します。POファイルは"src/docs/locale"に出力されます。

    sphinx-intl update -p _build/gettext -l ja

    POファイルは手作業で翻訳します。POファイルを開き、各"msgstr"の後に翻訳したメッセージを追加します。

    手作業での翻訳後、次のコマンドを実行してください。

    make -e SPHINXOPTS="-D language='ja'" html

    もしドキュメントを更新したら、sphinx-itl update コマンドでPOファイルを更新できます。

    sphinx-intl update -p _build/gettext
